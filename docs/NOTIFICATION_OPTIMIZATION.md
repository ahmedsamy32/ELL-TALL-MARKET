# تحسين نظام الإشعارات - Notification Optimization

## المشكلة الأصلية
كان النظام يرسل **6 إشعارات لكل طلب**، مما يسبب إزعاج للمستخدمين:
1. pending → confirmed
2. confirmed → preparing
3. preparing → ready
4. ready → in_transit
5. in_transit → delivered
6. إشعار إضافي عند التأكيد

## الحل المطبق

### 1. دمج الإشعارات الذكي (Smart Notification Merging)

تم دمج الإشعارات المرتبطة في إشعار واحد:

#### الدمج الأول: تأكيد + تحضير
```dart
case 'confirmed':
  title = 'تم قبول طلبك وجاري تحضيره ✅👨‍🍳';
  statusMessage = 'تم تأكيد طلبك من [المتجر] وبدأ التحضير الآن';

case 'preparing':
  // يتم تخطي هذا الإشعار في المستويات important و deliveryOnly
  return true; // تم دمجه مع confirmed
```

#### الدمج الثاني: جاهز + في الطريق
```dart
case 'ready':
  title = 'طلبك جاهز والكابتن في الطريق 📦🚗';
  statusMessage = 'طلبك جاهز والكابتن في طريقه إليك الآن';

case 'in_transit':
case 'on_the_way':
  // يتم تخطي هذا الإشعار في المستويات important و deliveryOnly
  return true; // تم دمجه مع ready
```

### 2. إعدادات مستويات الإشعارات (User Notification Preferences)

تم إضافة 3 مستويات للمستخدم للتحكم في عدد الإشعارات:

#### المستوى 1: كل التحديثات (All Updates)
- **عدد الإشعارات**: 4 إشعارات مدمجة
- **الإشعارات المرسلة**:
  1. تم قبول طلبك وجاري تحضيره ✅👨‍🍳
  2. طلبك جاهز والكابتن في الطريق 📦🚗
  3. تم التوصيل 🎉
  4. إلغاء (إذا تم الإلغاء) ❌

#### المستوى 2: المهم فقط (Important Only) - الافتراضي
- **عدد الإشعارات**: 3 إشعارات مدمجة
- **الإشعارات المرسلة**:
  1. تم قبول طلبك وجاري تحضيره ✅👨‍🍳
  2. طلبك جاهز والكابتن في الطريق 📦🚗
  3. تم التوصيل 🎉
- **يتم تخطي**: pending

#### المستوى 3: التوصيل فقط (Delivery Only)
- **عدد الإشعارات**: 2 إشعارات فقط
- **الإشعارات المرسلة**:
  1. تم التوصيل 🎉
  2. إلغاء (إذا تم الإلغاء) ❌
- **يتم تخطي**: كل الحالات الوسطية

## الملفات المعدلة

### 1. `lib/screens/user/notification_settings_screen.dart`
**التعديلات:**
- إضافة `enum NotificationLevel { all, important, deliveryOnly }`
- إضافة `SharedPreferences` للحفظ المستمر
- إضافة واجهة `RadioListTile` لاختيار المستوى
- إضافة أيقونات ووصف لكل مستوى
- حفظ وتحميل التفضيلات تلقائياً

**الكود الجديد:**
```dart
_buildNotificationLevelTile(
  title: 'المهم فقط (موصى به)',
  subtitle: 'إشعارات ذكية مدمجة (3 إشعارات)',
  value: NotificationLevel.important,
  icon: Icons.notifications,
)
```

### 2. `lib/services/notification_service.dart`
**التعديلات:**
- إضافة `import 'package:shared_preferences/shared_preferences.dart'`
- تعديل `notifyClientOfOrderStatusChange()` لتطبيق الدمج والتصفية
- إضافة `_shouldSkipNotification()` لتحديد الإشعارات المراد تخطيها
- دمج رسائل الإشعارات في رسائل أكثر ذكاءً

**المنطق الجديد:**
```dart
// تحميل التفضيلات
final prefs = await SharedPreferences.getInstance();
final notificationLevel = prefs.getInt('notification_level') ?? 1;

// تحديد ما إذا كان يجب التخطي
if (_shouldSkipNotification(newStatus, notificationLevel)) {
  return true; // تم التخطي
}

// دمج الإشعارات
if (status == 'preparing' && notificationLevel >= 1) {
  return true; // تم دمجها مع confirmed
}
```

## الاختبار

### اختبار المستوى "المهم فقط" (Important)
1. افتح **الإعدادات** → **إعدادات الإشعارات**
2. اختر "المهم فقط (موصى به)"
3. قم بعمل طلب جديد
4. تحقق من استلام 3 إشعارات فقط:
   - تأكيد + تحضير
   - جاهز + في الطريق
   - تم التوصيل

### اختبار المستوى "التوصيل فقط" (Delivery Only)
1. افتح **الإعدادات** → **إعدادات الإشعارات**
2. اختر "التوصيل فقط"
3. قم بعمل طلب جديد
4. تحقق من استلام إشعار واحد فقط عند التوصيل

### اختبار المستوى "كل التحديثات" (All)
1. افتح **الإعدادات** → **إعدادات الإشعارات**
2. اختر "كل التحديثات"
3. قم بعمل طلب جديد
4. تحقق من استلام 4 إشعارات مدمجة

## التطويرات المستقبلية المقترحة

1. **جدولة الإشعارات**
   - عدم الإزعاج في ساعات معينة
   - تأجيل الإشعارات غير الهامة

2. **إشعارات مخصصة حسب نوع الطلب**
   - إشعارات مختلفة للطعام مقابل البقالة
   - إشعارات خاصة للطلبات العاجلة

3. **إحصائيات الإشعارات**
   - معدل فتح الإشعارات
   - أكثر الإشعارات استجابة

4. **إشعارات تفاعلية**
   - "تم الاستلام" من الإشعار مباشرة
   - "تقييم الطلب" من الإشعار

## الخلاصة

تم تحسين نظام الإشعارات بنجاح من **6 إشعارات مزعجة** إلى **3-2 إشعارات ذكية ومدمجة**، مع إعطاء المستخدم التحكم الكامل في مستوى التفاصيل التي يريد استقبالها. هذا التحسين يرفع تجربة المستخدم بشكل كبير ويقلل من الإزعاج دون فقدان أي معلومات مهمة.
