# ميزة اختيار عنوان المتجر من الخريطة

## 📍 الميزة الجديدة

تم إضافة إمكانية اختيار عنوان المتجر من الخريطة بدلاً من إدخاله يدوياً، مما يضمن:
- ✅ دقة العنوان والإحداثيات
- ✅ حفظ المدينة والمحافظة تلقائياً
- ✅ التحقق الصحيح من نطاق التوصيل

## 🗂️ التحديثات المطلوبة

### 1. تحديث قاعدة البيانات

يجب تطبيق ملف الـ Migration التالي على قاعدة البيانات Supabase:

```
supabase/migrations/20241211_add_city_governorate_to_stores.sql
```

**خطوات التطبيق:**

1. افتح لوحة تحكم Supabase
2. اذهب إلى **SQL Editor**
3. افتح ملف `20241211_add_city_governorate_to_stores.sql`
4. انسخ المحتوى وألصقه في محرر SQL
5. اضغط **Run**

أو باستخدام Supabase CLI:
```bash
supabase db push
```

### 2. الملفات المحدثة

تم تحديث الملفات التالية:

#### Models:
- ✅ `lib/models/store_model.dart`
  - إضافة حقول: `city`, `governorate`
  - تحديث: `fromMap`, `toJson`, `toDatabaseMap`, `copyWith`

#### Services:
- ✅ `lib/services/store_service.dart`
  - تحديث `updateStoreFieldsV2` لحفظ المدينة والمحافظة

#### Screens:
- ✅ `lib/screens/merchant/merchant_settings_screen.dart`
  - إضافة متغيرات: `_storeCity`, `_storeGovernorate`, `_storeLatitude`, `_storeLongitude`
  - إضافة دالة: `_pickAddressFromMap()`
  - تحديث واجهة العنوان مع زر الخريطة

## 🎨 كيفية الاستخدام

### للتاجر:

1. اذهب إلى **إعدادات المتجر**
2. في قسم **معلومات التواصل**
3. بجانب حقل العنوان ستجد أيقونة **خريطة** 🗺️
4. اضغط على الأيقونة لفتح الخريطة
5. اختر الموقع الدقيق للمتجر
6. سيتم ملء العنوان والمدينة والمحافظة تلقائياً
7. اضغط **حفظ التغييرات**

### عرض البيانات:

بعد اختيار العنوان من الخريطة، سيظهر تأكيد أخضر يحتوي على:
```
✓ المدينة، المحافظة
مثال: ✓ التل الكبير، الإسماعيلية
```

## 🔄 التكامل مع نظام التوصيل

الآن عند إضافة طلب:

1. **العميل** يختار عنوان التوصيل من الخريطة
   - يحصل على: `city`, `governorate`

2. **النظام** يتحقق من جميع المتاجر في السلة
   - يقارن `عنوان العميل` مع `عنوان كل متجر`
   - إذا كان المتجر في مدينة/محافظة مختلفة → **خارج نطاق التوصيل**

3. **التحذير** يظهر قائمة بالمتاجر خارج النطاق
   - يمنع إتمام الطلب
   - يطلب من العميل اختيار عنوان آخر أو حذف المنتجات

## 📊 بنية البيانات

### جدول `stores`:

```sql
city         TEXT         -- المدينة (مثال: التل الكبير)
governorate  TEXT         -- المحافظة (مثال: الإسماعيلية)
latitude     DECIMAL(10,8)  -- خط العرض
longitude    DECIMAL(11,8)  -- خط الطول
```

### فهارس الأداء:

```sql
idx_stores_city              -- للبحث بالمدينة
idx_stores_governorate       -- للبحث بالمحافظة
idx_stores_city_governorate  -- للبحث بالاثنين معاً
```

## ⚠️ ملاحظات مهمة

1. **للمتاجر الموجودة**: يجب على التجار تحديث عناوينهم من الخريطة
2. **التوافق العكسي**: المتاجر بدون city/governorate ستظل تعمل لكن بدون تحقق من نطاق التوصيل
3. **الإحداثيات**: يتم حفظها للاستخدام المستقبلي (حساب المسافة، خرائط Google، إلخ)

## 🐛 استكشاف الأخطاء

### المشكلة: "لا يمكن حفظ العنوان"
**الحل**: تأكد من تطبيق ملف الـ Migration أولاً

### المشكلة: "لا يظهر زر الخريطة"
**الحل**: تأكد من تحديث ملف `merchant_settings_screen.dart`

### المشكلة: "التحقق من نطاق التوصيل لا يعمل"
**الحل**: تأكد من أن المتجر قام بتحديث عنوانه من الخريطة
